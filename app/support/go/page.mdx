# GitHub Offboarding Support

## Removing a GitHub Organization from Hellō

To fully remove a GitHub organization from Hellō, complete both steps below.

### Step 1: Delete the Organization from the Hellō Dashboard

1. Go to your [Hellō GitHub Offboarding dashboard](https://lifecycle.hello.coop/github).
2. Click **Delete** next to the organization you want to remove.
3. Type `DELETE` to confirm.

This removes the connection configuration, account-linking data, and the `Hello-Lifecycle` repository from your GitHub organization. Automated offboarding will immediately stop working for this org.

> If you are removing the entire organization from Hellō Lifecycle (not just GitHub Offboarding), see [Removing an Organization from Hellō](/support/lifecycle#removing-an-organization-from-hellō).

### Step 2: Uninstall the Hellō GitHub App

The Hellō Lifecycle GitHub App remains installed on your organization until you manually remove it.

1. Go to your organization's GitHub App settings:
   ```
   https://github.com/organizations/<YOUR_ORG_NAME>/settings/installations
   ```
2. Find **Hellō Lifecycle** and click **Configure**.
3. Scroll to the bottom and click **Uninstall**.

## Troubleshooting

### GitHub Organization Not Showing Up During Setup

When setting up Hellō Lifecycle at [lifecycle.hello.coop/github](https://lifecycle.hello.coop/github), you may find that your organization doesn't appear in the "Select Your Organization" step. This can happen for two different reasons:

#### Reason 1: You Skipped Granting Access During Installation

During the GitHub OAuth app authorization flow, you may have skipped granting access to your organization.

**Solution:**

1. Go to the [Hellō Lifecycle OAuth app settings page](https://github.com/settings/connections/applications/lv23li3rZkORXbEiBVu9).

2. On this page, you'll see all the organizations you belong to.

3. Click **Grant** next to the organization you want to use with Hellō Lifecycle.

4. Return to [lifecycle.hello.coop/github](https://lifecycle.hello.coop/github), log out, and go through the GitHub authorization flow again.

#### Reason 2: Your Organization Has Disabled Third-Party Access

If you see a red cross beside your organization and the **Grant** button is not visible (either during installation or when visiting the OAuth app settings page above), this means someone in your organization has disabled third-party OAuth app access for Hellō Lifecycle.

**Solution:**

1. Go to your organization's OAuth application policy settings:
   ```
   https://github.com/organizations/<YOUR_ORG_NAME>/settings/oauth_application_policy
   ```
   Replace `<YOUR_ORG_NAME>` with your organization's name (e.g., `https://github.com/organizations/Bubbler/settings/oauth_application_policy`).

2. Find **Hellō Lifecycle** in the list of apps. It will have a **Denied** label beside it.

3. Click the **Edit** button next to Hellō Lifecycle.

4. Click **Grant access**.

5. Return to [lifecycle.hello.coop/github](https://lifecycle.hello.coop/github), log out, and go through the GitHub authorization flow again.

> **Note:** You will need organization admin privileges to change OAuth application policies.
